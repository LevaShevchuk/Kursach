Контекстные ключевые слова

Контекстные ключевые слова используются для предоставления конкретного значения в программном коде, но они не являются зарезервированными словами в языке C#.

Глоссарий контекстных ключевых слов


add — определяет метод доступа настраиваемого события, который вызывается, когда клиентский код подписывается на событие.

and — логическое И.

аlias — создаёт псевдоним для полной команды, полной команды с аргументами или для другого псевдонима.

аscending — используется в предложении orderby в выражении запроса, задавая порядок сортировки от наименьшего значения к наибольшему.

аsync — позволяет указать, что метод, лямбда-выражение или анонимный метод является асинхронным.

аwait — приостанавливает вычисление включающего асинхронного метода до завершения асинхронной операции, представленной его операндом.

by — используется в предложении group в выражении запроса для определения способа группировки возвращаемых элементов.

descending — используется в предложении orderby в выражении запроса, задавая порядок сортировки от наибольшего значения к наименьшему.

dynamic — указывает, что использование переменной и ссылок на её члены обходит проверку типа во время компиляции.

equals — используется в предложении join в выражении запроса для сравнения элементов двух последовательностей.

from — определяет источник данных, в отношении которого будет выполнен запрос или вложенный запрос. Также контекстное ключевое слово from определяет локальную переменную диапазона, которая представляет каждый элемент в последовательности источника.

get — определяет метод доступа в свойстве или индексаторе, который возвращает значение свойства или элемент индексатора.

global — является псевдонимом глобального пространства имён.

group — возвращает последовательность объектов IGrouping<TKey,TElement>, содержащих ноль или более элементов, соответствующих значению ключа группы.

init — определяет метод доступа в свойстве или индексаторе.

into — используется для создания временного идентификатора для сохранения результатов предложений group, join или select в новый идентификатор.

join — используется для связывания элементов из разных исходных последовательностей, не имеющих прямых связей в объектной модели.

let — используется для сохранения результата вложенного выражения, который будет использоваться в последующих предложениях.

nameof — создаёт имя, тип или элемент переменной в качестве строковой константы.

nint — используется для определения целых чисел.

not — логическое НЕ.

notnull — используется для того, чтобы указать, что аргумент типа должен быть типом значения, не допускающим значения NULL, или ссылочным типом, не допускающим значения NULL.

nint (nuint) — используется для определения целых чисел собственного размера.

on — используется в предложении join выражения запроса, задавая условие объединения.

orderby — задаёт сортировку возвращаемой последовательности или вложенной последовательности (группы) в порядке возрастания или убывания.

partial — указывает, что другие части класса, структуры или интерфейса могут быть определены в пространстве имён.

record — класс, предоставляющий специальный синтаксис и поведение для работы с моделями данных.

remove — определяет метод доступа настраиваемого события, который вызывается, когда клиентский код отменяет подписку на событие.

select — задаёт тип значений, которые будут получены при выполнении запроса.

set — определяет метод доступа в свойстве или индексаторе, который присваивает значение свойству элемента индексатора.

unmanaged — используется для того, чтобы указать, что параметр типа должен быть неуправляемым типом, не допускающим значения NULL.

value — используется в методе доступа set в объявлениях свойства и индексатора.

var — ссылается на тип неявным способом.

when — используется для указания условия фильтра в последующих контекстах.

where — задаёт ограничения на типы, которые используются в качестве аргументов для параметров типа в универсальном типе, методе, делегате или локальной функции.

with — создаёт копию операнда record с изменёнными заданными свойствами и полями.

yield — используется для обозначения, что метод, оператор или метод доступа get, в котором присутствует это ключевое слово, является итератором.
